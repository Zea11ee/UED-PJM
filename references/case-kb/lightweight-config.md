# 模板：轻量配置/联调

## 适用信号
- 工时总量小（常见 <= 2h 或 <= 1d）。
- 主要是配置项调整、文案/样式小改、已存在能力复用。
- 需求目标清晰，不需要完整开发闭环。

## 推荐任务链（最小集）
1. 产品配置/PRD（可选，0.5h~2h）
2. 开发或配置执行（0.5h~4h）
3. 走查/验收（0.5h~2h）

## 常见命名
- `产品配置`
- `开发联调` / `开发实现`
- `测试与回归` / `上线走查`

## 排期关键点
- 避免过度拆分，优先同日完成。
- 若有上线节点，走查任务必须绑定上线当天。
- 先去重（按源编号），避免反复创建父任务。
